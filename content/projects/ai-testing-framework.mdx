## Overview

A detailed write-up of this framework is in progress. Below is the shape of what's there and what I'm comfortable discussing publicly.

## The problem

AI-native features fail differently from deterministic code. The output is "wrong" in subjective ways, model providers ship breaking behavior changes without bumping a version, and naive end-to-end tests either pass everything (overly lenient assertions) or fail every run (assertion brittleness). Teams shipping these features need a different kind of harness.

## What's in the framework

- **Prompt regression suites.** Frozen inputs, scored outputs, threshold-based pass/fail with human-readable diffs when a score drops.
- **Model-versus-model evaluation.** Runs a candidate model and a baseline model across the same input set, surfaces deltas worth reviewing.
- **Hallucination guardrails.** Output-shape and grounding checks plugged into GitHub Actions so they run on every PR that touches an AI surface.
